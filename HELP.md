# 方案策划专家 (solution-planning-expert) — 使用帮助文档

> 版本：v2.0 | 更新时间：2026-07-02 | 位置：`~/.workbuddy/skills/solution-planning-expert/`

---

## 1. 技能概述

### 定位
智能制造解决方案助手。读取客户招标需求文档后，从需求解析→产品能力匹配→方案大纲生成→最终输出可编辑PPT的端到端自动化方案策划工具。

### 适用场景
- 客户提供了 Word/Excel/PDF/Markdown 格式的招标需求或技术规格书
- 需要基于盘古 IMS-V6（BS+MES+QCS+EAM+WMS+IOT+STS）产品体系输出方案
- 需要生成结构化的述标 PPT（35-50页，含封面/章节分隔/内容/封底）
- 需要引用 IMA 知识库中的真实产品操作手册作为方案依据

### 触发方式
对话中上传需求文件并提及以下关键词即可激活：
> 「写方案」「生成方案」「方案策划」「项目方案」「解决方案框架」「输出PPT」

## 3. 工作流详解（6步）

### 第1步：需求解析与展示
**目标**：将客户需求文档结构化拆解为五大维度。

**输入**：Word(.docx) / Excel(.xlsx) / PDF(.pdf) / Markdown(.md) 需求文档

**处理逻辑**：
1. 根据文件类型选择读取工具（pandas/openpyxl → Excel；python-docx → Word；pdfplumber → PDF；直接读取 → Markdown）
2. 按五大维度拆解：
   - **一、业务范围**：客户名称 / 行业 / 主要产品 / 生产模式 / 工厂规模
   - **二、功能需求**：一级功能 → 二级功能 → 具体需求描述（逐条罗列）
   - **三、项目集成需求**：对接系统 / 集成方式 / 数据交互说明
   - **四、核心需求**：客户最关注的3-5条核心痛点
   - **五、设备集成管理**：设备清单 / 数量 / 通信协议 / 数据采集要求
3. 逐项检查是否有缺失，**缺失项必须向用户提问**，待补充后继续

**输出**：结构化需求拆解表格（对话内呈现）

**参考模板**：`references/requirement-template.md`

---

### 第2步：IMA知识库检索 🆕
**目标**：从 IMA 知识库中提取产品真实能力，为后续映射提供产品依据。

**输入**：无需额外输入（自动检索用户的知识库）

**处理逻辑**：
1. 调用 `mcp__ima-mcp__get_knowledge_base_list` 获取知识库列表，定位包含产品操作手册的知识库（如「蒋俊涛Tower的知识库」）
2. 调用 `mcp__ima-mcp__search_knowledge` 搜索：
   - 关键词："产品操作手册" / "功能说明文档"
   - 产品名：IMS-MES / IMS-QCS / IMS-EAM / IMS-WMS / IMS-IOT / IMS-STS / IMS-BS
3. 调用 `mcp__ima-mcp__fetch_media_content` 读取七产品功能说明文档
4. 提取关键产品能力：
   - **IOT**：设备驱动数量与类型、支持协议（OPC UA/Modbus/MQTT）、CIMS框架
   - **STS**：ERP品牌适配范围（SAP/用友/金蝶/Oracle-EBS）、队列合并、多实例路由
   - **BS**：低代码配置领域、三员分立权限、三策略体系
   - **MES/QCS/EAM/WMS**：各产品核心模块与功能清单
   - **基础设施**：高可用（LVS+keepalived）、双数据库（PostgreSQL/Oracle）、Cloud Label 条码、企微/钉钉集成

**输出**：七产品核心能力摘要（注入后续映射）

---

### 第3步：系统功能映射（增强版）🆕
**目标**：将客户每项功能需求映射到六系统之一，并标注产品依据。

**输入**：第1步的需求拆解 + 第2步的产品能力

**处理逻辑**：
1. 逐条遍历需求，从**仅限的6个系统**中选择负责系统：
   - WMS（仓库管理）、MES（制造执行）
   - QCS（品质管理）、EAM（设备管理）
   - IOT（设备联网平台）、STS（系统集成平台）
2. 生成4列映射表（新增"产品依据"列）：

   | 序号 | 需求功能描述 | 负责系统 | 产品依据（IMA文档） | 简要实现说明 |
   |------|-------------|---------|-------------------|-------------|

3. 无法覆盖的需求标注"超出系统范围"并给出建议

**输出**：功能-系统映射表（Markdown表格，含产品依据列）

---

### 第4步：联网检索与方案大纲生成
**目标**：结合行业背景和产品能力，生成PPT格式的方案大纲。

**输入**：第3步的映射结果

**处理逻辑**：
1. **联网检索**：`WebSearch` 搜索客户公司背景/行业趋势/技术案例（注明来源）
2. **生成大纲**：按6章结构输出，每页遵循 PPT 格式模板：

| 章节 | 页数 | 内容 |
|------|------|------|
| 项目理解与方案定位 | 5-8页 | 执行摘要/战略定位/行业洞察/项目范围/业务全景/架构愿景 |
| 整体技术方案 | 8-12页 | BS底座/IOT/STS/HA/主数据/低代码/执行/ANDON/移动端 |
| 核心业务场景 | 10-15页 | 生产工艺/特殊工序/定制业务/质量/设备/仓储/追溯/人员/工艺/包装 |
| 业财一体化 | 5-10页 | 架构/直接成本/分摊/WIP/返工/损耗/薪资结案/审计/经营报表 |
| 实施与保障 | 4-6页 | 路线图/组织/风险/扩展/运维/演进 |
| 商业价值与总结 | 3-5页 | 效率/损耗/ROI/案例/总结/封底 |

3. **合并原则**：相似主题合并一页、每页4-6个要点、核心场景独立展开、目标总35-50页

**输出**：`*_述标PPT大纲_V2.0.md` 文件（按 `references/ppt-format.md` 格式，每页含标题/关键字/要点/数据/视觉/演讲词）

**参考模板**：`references/ppt-format.md`

---

### 第5步：PPT方案输出 🆕
**目标**：将大纲转换为可编辑的 `.pptx` 文件。

**输入**：第4步输出的 Markdown 大纲文件

**处理逻辑**：
1. 调用 `Skill` 加载 `pptx` 技能
2. 使用 `pptxgenjs` (Node.js) 从零生成 PPT：
   - 布局：16:9 (10"×5.625")
   - 字体：Arial Black（标题）+ Arial（正文）
   - 配色：从大纲各页"视觉元素→颜色基调"提取
3. 四类幻灯片：
   - **封面**：深色背景 + 大标题 + 副标题 + 公司信息
   - **章节分隔页**：深色背景 + 章节编号(72pt) + 章节名 + 标题
   - **内容页**：浅灰背景(#F0F4F8) + 左侧0.08"teal accent bar + 顶部深蓝细条 + 标题(20pt Arial Black) + gold下划线 + 要点列表 + 页脚(公司/项目/页码)
   - **封底**：暗色背景 + 核心主张 + 六大优势卡片(2×3 grid)

**输出**：`*.pptx` 文件

**设计约束**：
- 颜色使用6位hex不带`#`前缀（`"065A82"` ✓ | `"#065A82"` ✗）
- 阴影opacity独立属性，不编码到颜色中
- 每个 `addShape` options 对象全新创建
- 使用 `bullet: true`，不用 Unicode 符号

---

### 第6步：持续澄清
**贯穿全流程**：任一阶段发现信息不足以支撑准确内容时，立即中断、提问、等待回复、再继续。

**典型触发场景**：
- 客户名称不明确 → "请确认客户公司全称？"
- 设备协议不明确 → "设备品牌和通信协议是什么？"
- 集成系统不明确 → "需要对接哪些ERP系统？"
- 工厂规模不明确 → "产线数量和年产能是多少？"

## 4. 系统约束

### 系统白名单（不可扩展）

| 缩写 | 系统 | 覆盖领域 |
|------|------|---------|
| WMS | 仓库管理系统 | 投料/齐套/入库/退料/形态库存 |
| MES | 制造执行系统 | 生产执行/计划/追溯/ANDON/BI/人员 |
| QCS | 品质管理系统 | IQC→FQC全阶/NCR+8D/SPC |
| EAM | 设备管理系统 | 全生命周期/维保/OEE/备件/工装 |
| IOT | 设备联网平台 | 驱动(94种)/CIMS/协议/数采 |
| STS | 系统集成平台 | ERP集成/队列合并/多实例路由/消息推送 |

### 禁止使用/提及的系统
ERP、PLM、SRM、CRM、HR、TMS、EMS 及任何不在白名单内的系统。

> **例外**：当客户明确提到这些系统时，可作为集成对接对象在 STS 方案中提及，但不可将其作为方案设计的负责系统。

## 7. PPT设计规范速查

### 7.1 配色方案（Ocean Gradient）

| 用途 | 颜色名 | Hex值 |
|------|--------|-------|
| 封面/章节/封底背景 | Midnight | `21295C` |
| 标题/顶部条 | Deep Blue | `065A82` |
| 左侧accent bar/章节编号 | Teal | `1C7293` |
| 标题下划线/高亮 | Gold | `D69E2E` |
| 内容页背景 | Light Gray | `F0F4F8` |
| 正文文字 | Dark Text | `1E293B` |
| 页脚文字 | Muted | `64748B` |

### 7.2 幻灯片尺寸与字体

| 元素 | 规格 |
|------|------|
| 布局 | `LAYOUT_16x9` (10" × 5.625") |
| 标题字体 | Arial Black, 20pt (内容页) / 36pt (封面) |
| 正文字体 | Arial, 11-12pt |
| 章节编号 | Arial Black, 72pt |

### 7.3 pptxgenjs 常见陷阱

| 陷阱 | 正确做法 |
|------|---------|
| 颜色带 `#` 前缀 | 只用6位hex: `"FF0000"` |
| 阴影颜色含透明度 | 独立 `opacity` 属性 |
| 复用 options 对象 | 每次 `addShape` 全新创建 |
| Unicode 符号 `"•"` | 使用 `bullet: true` |
| `ROUNDED_RECTANGLE` + accent bar | 改用 `RECTANGLE` |

## 8. 完整使用示例

### 示例1：端到端方案生成

```
用户：@"D:/招标需求/华鲲MES招标需求.xlsx" 需求解析与展示

技能执行：
  → 第1步：读取Excel → 结构化拆解 → 展示五大维度 → 提问澄清缺失项
  → 用户补充回答
  → 第2步：检索IMA知识库 → 获取七产品能力
  → 第3步：生成52项功能-系统映射表（含产品依据列）
  → 第4步：WebSearch华鲲背景 → 生成50页PPT格式方案大纲
  → 用户确认大纲
  → 第5步：调用pptx技能 → pptxgenjs生成PPT → QA验证
  → 输出：华鲲MES述标方案.pptx
```

### 示例2：快速大纲输出

```
用户：整理华鲲MES项目方案大纲，相似内容整合，50页左右，按参考格式输出

条件：之前已完成需求解析和系统映射

技能执行：
  → 第4步：根据已有映射结果生成50页PPT格式大纲
  → 输出：华鲲MES述标PPT大纲_V2.0.md
```

### 示例3：大纲转PPT

```
用户：@"/path/华鲲MES述标PPT大纲_V2.0.md" 根据大纲内容，输出PPT

技能执行：
  → 第5步：读取MD大纲 → pptxgenjs生成56页PPT
  → 输出：华鲲MES述标方案_盘古IMS-V6.pptx (761KB)
```

### 示例4：单独产品能力查询

```
用户：检索IMA知识库中产品操作手册的内容，进行系统功能映射
```

## 9. 设计决策记录

| 决策 | 理由 |
|------|------|
| 六系统白名单不可扩展 | 对标盘古IMS-V6产品体系，方案不涉及非自有系统 |
| 映射表新增"产品依据"列 | 引用IMA真实文档名称，提升方案可信度和专业感 |
| 大纲按PPT页面格式而非表格输出 | 直接对接第5步PPT生成，减少中间转换 |
| 目标页数35-50页 | 述标PPT标准时长45分钟，过多超出注意力时长，过少信息密度不足 |
| 第2步独立于第1步 | IMA检索需要MCP工具支持，与需求解析技术栈分离 |
| 内容页4-6个要点 | 少于4点单薄，多于6点拥挤，4-6点为最佳平衡点 |
